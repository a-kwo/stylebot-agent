# Improving StyleBot's Preference Understanding & Recommendation Quality

## Context
StyleBot collects reasonably rich preference data (V2 style vector, weighted adjectives, feedback, wardrobe) but underutilizes it. The style vector is rendered as raw floats Claude must interpret. Feedback stores only product titles. Search queries are entirely Claude-improvised with no programmatic guardrails. Preferences are additive-only and never decay. The result: recommendations feel generic rather than deeply personalized.

---

## Current Methodology vs. Proposed Improvements

### 1. Style Vector -> Search Guidance (Tier 1 - High Impact, Low Complexity)

Current: Claude sees `energy: 0.60, cultural_ref.sport_street: 0.75, color.temperature: 0.15` -- raw numbers with labels like "balanced" or "Sport/Street". Claude must guess what this means for product searches.

Proposed: A deterministic translation layer that maps vector dimensions to concrete product attributes -- brand suggestions, search keywords, material preferences, aesthetic references.

Example output Claude would see:
  Search Guidance (from style profile):
  - Brands to explore: Nike, Stussy, Carhartt WIP, New Balance
  - Keywords: clean streetwear, minimal sneakers, relaxed joggers
  - Materials: cotton jersey, nylon, canvas
  - Avoid in searches: formal suiting, dress shoes, silk

Files: New `services/style_translator.py`, modify `agent/system_prompt.py`

---

### 2. Enriched Feedback Storage (Tier 1 - High Impact, Low Complexity)

Current: `recommendation_feedback` stores only `product_title` + `"like"/"dislike"`. Claude sees a flat list of titles and must parse patterns from text alone.

Proposed: Store product metadata alongside feedback -- `price`, `category`, `seller`, `color`, `search_query`, `image_url`. Render aggregate patterns in the system prompt.

Example Claude would see:
  Feedback Patterns (last 20):
  - Price sweet spot: $40-80 (4/5 likes in this range)
  - Preferred sellers: Nike, ASOS (3 likes each)
  - Avoided patterns: floral, graphic print (2 dislikes)
  - Category interest: outerwear (3 likes), shoes (2 likes)

Files: `database.py` (add columns), `models.py`, `routers/chat_router.py`, `agent/system_prompt.py`

---

### 3. Programmatic Search Query Enhancement (Tier 1 - High Impact, Low Complexity)

Current: Only gender is auto-prepended to queries. Budget, avoided brands, preferred brands, sizes, and fit preferences are left entirely to Claude's discretion. Claude often forgets budget params.

Proposed: A `_enhance_query()` function that auto-injects profile data:
- Always set `min_price`/`max_price` from profile budget if Claude omitted them
- Append preferred brand names for generic queries
- Warn if query contains an avoided brand

Files: `agent/tool_handlers.py`

---

### 4. Negative Signal Filtering (Tier 2 - High Impact, Medium Complexity)

Current: Avoided colors/brands are listed in the system prompt but never enforced. Claude sometimes recommends items in avoided colors when SerpAPI returns them prominently.

Proposed: Post-filter SerpAPI results to remove products matching avoided brands/colors (case-insensitive title match). Re-search with exclusion terms if too many results filtered out.

Files: `agent/tool_handlers.py`, `services/shopping.py`

---

### 5. Behavioral Reflection Upgrade (Tier 2 - High Impact, Medium Complexity)

Current: Reflection prompt extracts only explicit signals (style adjectives, colors, brands). Instructed to "be conservative -- only extract clear signals, not guesses." Misses price sensitivity, category engagement, and implicit reactions.

Proposed: Expand reflection to detect:
- Price reactions ("that's a bit much" -> price sensitivity)
- Category interest (repeated questions about outerwear -> strong interest)
- Engagement level (long discussion = high interest, quick dismissal = low)
- Pass recent feedback into reflection for correlation

Files: `agent/loop.py` (reflection prompt + handler), `agent/system_prompt.py`

---

### 6. Wardrobe-Aware Style Coherence Analysis (Tier 2 - Medium Impact, Medium Complexity)

Current: `analyze_wardrobe` returns category counts, missing categories, flat color list. No cross-referencing with profile or style vector.

Proposed: Richer analysis including:
- Style vector alignment ("wardrobe is 80% casual but you work 3 days/week")
- Color temperature analysis (warm vs cool vs neutral distribution)
- Occasion coverage vs stated occasions
- Specific gap recommendations informed by style vector

Files: `agent/tool_handlers.py`

---

### 7. Preference Decay & Confidence Scoring (Tier 3 - Medium Impact, Medium-High Complexity)

Current: Array fields (colors, brands, occasions, fits) are union-merged and never removed. A preference from months ago persists forever even if contradicted by recent behavior.

Proposed: Migrate array fields to weighted format (like `style_adjectives` already uses). Add time-decay so old unreinforced preferences fade. Add decrement capability so negative signals reduce weights.

Files: `services/profile_service.py`, `database.py`, `agent/system_prompt.py`, `agent/tools.py`

---

### 8. Feedback-Driven Style Vector Refinement (Tier 3 - Medium Impact, Medium-High Complexity)

Current: Style vector is set once during onboarding quiz and never updated. A user who quizzed as "prep" but keeps liking streetwear stays frozen as "prep."

Proposed: After every N feedback signals, compute a `feedback_vector` from product attributes and blend: `refined = 0.7 * quiz + 0.3 * feedback`. Store both original and refined vectors.

Files: New `services/vector_refinement.py`, `agent/loop.py`, `database.py`

---

### 9. Post-Search Product Re-ranking (Tier 3 - Medium Impact, Medium-High Complexity)

Current: Products returned in SerpAPI's generic relevance order. No personalization of result ordering.

Proposed: Deterministic scoring pass after SerpAPI returns: brand match (+/-), price fit, color alignment, wardrobe novelty (don't recommend duplicates), seller diversity. Sort by composite score.

Files: New `services/product_ranker.py`, `agent/tool_handlers.py`

---

### Future Ideas (Tier 4 - requires scale/data)
- Contextual preference modeling: Preferences tagged by occasion (bold for nightlife, neutral for work)
- Engagement tracking: Time spent, clicks, saves -> behavioral interest model
- Collaborative filtering: Similar user recommendations (requires user volume)

---

## Recommended Implementation Order

Phase 1 (items 1-3): Immediate wins, minimal risk. Style translator, enriched feedback, query enhancement.

Phase 2 (items 4-6): Capability upgrades. Negative filtering, behavioral reflection, wardrobe coherence.

Phase 3 (items 7-9): System maturity. Preference decay, vector refinement, re-ranking.

## Verification
- After each phase, test end-to-end: complete onboarding quiz -> chat -> request suggestions -> like/dislike products -> request more suggestions -> verify recommendations improve
- Check system prompt output to confirm new sections render correctly
- Test edge cases: empty profile, no feedback history, avoided brand appearing in results
